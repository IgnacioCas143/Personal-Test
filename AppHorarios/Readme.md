## About the Project

This application allows users to create and manage a shared schedule for multiple team members.
Each member can be assigned classes or activities with customizable colors and visual patterns, making it easy to identify overlaps and availability.

The goal of this project was to build a dynamic and interactive UI using only vanilla JavaScript, focusing on usability and visual clarity.
---
##  Technologies Used
* HTML5
* CSS3 (Grid, Flexbox, custom UI styles)
* JavaScript (DOM manipulation, events)
* LocalStorage (data persistence)
---
##  Features
* Dynamic schedule grid (days & hours)
* Team member management (add, edit, delete)
* Custom colors and patterns per member
* Create, edit, and delete classes
* Filter by member or day
* Persistent data using LocalStorage
* Heatmap visualization for schedule density(optional/disabled)
---
## How It Works
* The schedule is generated dynamically using a grid layout.
* Each class is rendered as a block positioned based on time and day.
* Members are visually differentiated using colors and patterns.
* Data is stored locally, so the schedule persists after refreshing.
